## Executive Summary

The repo is a port-in-progress from `prototype` (RAG era) to the EAG paradigm. Most subsystems (extraction, custodian, clustering, write-path, stores) are implemented and wired; the **MCP surface — the primary agent interface — is entirely stubbed (`NotImplementedError` in every tool)**. This single fact is the dominant operational risk and corrupts a number of secondary findings (auth, tenancy, perf targets are unverifiable until tools land). Apart from that, the substrate is in reasonable shape, with a recurring cluster of issues around (a) write-path atomicity in the custodian, (b) N+1 patterns in the extraction/clustering hot loops, (c) silently-defined-but-never-applied indexes for `:Cluster` / `:Entity(name)` / `:Document(source_uri)`, (d) auth middleware defined but not mounted, and (e) primitives `eag.epistemology` helpers not yet wired (consciously deferred per `eag-integration-audit.md`).

## Themes

1. **MCP surface is a paper tiger.** All five registered tools (`context_get`, `context_store`, `context_lookup`, `silo_create`, `silo_list`) raise `NotImplementedError`; three more tools documented in `context/api-examples.md` (`context_link`, `context_graph`, `context_store_chain`) plus all four Meta-Memory tools have no files at all. Tool signatures that *are* defined diverge from the documented contract (`silo_id` vs `silo_ids`, missing `org_id`, missing `tags`/`filters`). Findings: **F-001, F-018, F-019, F-020, F-021**.

2. **Custodian write-path atomicity is fragile.** Consensus promotion creates `:Finding` via `CREATE` (not `MERGE`) with a non-deterministic id (includes `datetime.now()`), runs across multiple un-transacted `execute_write` calls, and `execute_write` itself has no retry. Dagster resource teardown uses `asyncio.run()` inside an already-running loop, silently swallowed via `contextlib.suppress(RuntimeError)` — Memgraph/Qdrant/Redis drivers are never closed. Findings: **F-006, F-007, F-008, F-022**.

3. **Auth wiring is half-built.** `MCPAuthMiddleware` is defined and exported but never mounted in `create_app()`. When `MCP_API_KEY` is absent, callers are granted full admin (`["read","write","admin"]`, `org_id="dev-org"`) with no env guard. The `auth/` package (intended to host WorkOS) is empty. The `context_lookup` and `context_get` tool signatures both accept silo references without validating them against the auth-derived `org_id`. Findings: **F-009, F-010, F-011, F-012**.

4. **Indexes defined as constants but never applied.** `CREATE_CLUSTER_ID_INDEX`, `CREATE_CLUSTER_LEVEL_INDEX`, `CREATE_CLUSTER_SILO_INDEX` are duplicated across `db/queries.py` and `clustering/queries.py` but neither registers them in `db/indexes.ALL_INDEX_QUERIES`, so `FETCH_CLUSTERS_BY_LEVEL` does a full scan on every custodian pass. `:Entity(name)` and `:Document(source_uri)` are also unindexed despite being filtered in hot paths. Findings: **F-013, F-014, F-015**.

5. **N+1 hot loops in extraction & clustering.** `apply_claims_to_graph` / `apply_document_claims` issue 3+ Memgraph round-trips per claim (claim upsert + passage attach + per-mention upserts). `embed_cluster_summaries` batches the embedding API call but then makes per-cluster Qdrant upserts despite `batch_upsert` existing. `_link_hierarchy` and `build_hierarchy` write `MEMBER_OF` / `PART_OF` edges one at a time. Findings: **F-016, F-017, F-024, F-025**.

6. **EAG paradigm not fully realized.** `primitives.eag.epistemology` (combined_confidence, R1/R2 promotion, structural contradiction detection) is imported nowhere. Custodian implements a separate Finding-status flip (`promotion.py`) and routes contradiction detection through the LLM only. Wisdom layer (`:Belief`, `:Pattern`, `SYNTHESIZED_FROM`) has indexes but no write path. `:Fact` label has indexes but consensus promotion writes `:Finding` instead. Most of this is *consciously deferred* in `context/plans/eag-integration-audit.md` — but the deferred items are starting to compound. Findings: **F-002, F-003, F-004, F-005, F-026**.

7. **Spec invariants violated in the query layer.** CLAUDE.md rule 7 (`:Finding` reads must filter `source='extraction' OR status='published'`) is violated by `TOMBSTONE_DOCUMENT`, `CREATE_CRYSTALLIZED_INTO_EDGE`, `CREATE_DERIVED_FROM_EVIDENCE_EDGE`, `CHECK_CHAIN_AUDIT_LINKED`. The primitives `06-epistemology` I5 invariant (SUPERSEDES requires `reason`) is violated by `UPSERT_NODE_SINGLE_RTT` and `CREATE_CROSS_NODE_SUPERSEDES` (which write `source` but not `reason`). `CLUSTER_LAST_CUSTODIAN_UPDATE` lacks any `silo_id` predicate, allowing cross-tenant writes. Findings: **F-027, F-028, F-029**.

## Notes / Recommendations

1. **Land F-001 first.** Most security/perf findings are speculative until the MCP tools are implemented. The P1 auth/tenancy findings (F-009 through F-012) become P0 the moment the tools start returning data — fix them in the same sprint.
2. **Single transaction for consensus promotion (F-006).** This is the highest-priority correctness fix that doesn't depend on F-001. Concurrent Dagster runs *can* hit this today.
3. **Apply the cluster indexes (F-013).** Cheap, blocking nothing, immediate measurable win on the custodian pass.
4. **Author `context/review/invariants.md`.** Bake CLAUDE.md rule 7 and primitives I5 into a deterministic invariant check (Agent 7), so the next review enforces them mechanically. Suggested seed:
   - `finding_read_filter`: every `:Finding` MATCH must have `source='extraction' OR status='published'` predicate (verify via grep).
   - `supersedes_reason`: every `SUPERSEDES {...}` write must include a `reason` property.
   - `silo_scope`: every read/write Cypher in `db/` and `engine/queries.py` must reference `silo_id` in MATCH or WHERE.
5. **Track deferred EAG items as a single roadmap item.** F-002, F-004, F-005, F-026, F-056 all live in `eag-integration-audit.md` already — the review re-surfaces them only because they intersect spec invariants or have started accumulating dead schema (`:Fact`, `:Belief`, `:Pattern`, `:Utterance`, `:Event`, `:QueryContext` indexes without write paths).
