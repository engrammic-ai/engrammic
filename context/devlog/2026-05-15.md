## Summary

Shipped intent-based MCP tool surface (v1.3.1) and synced all documentation across repos and Notion.

## MCP Tool Surface Redesign

Replaced layer-based tools with intent-based naming for better agent discoverability.

**Old surface (10 tools):**
- `context_store`, `context_recall`, `context_link`, `context_admin`
- `context_belief_state`, `context_update_belief`, `context_crystallize`
- `context_accept_belief`, `context_reject_belief`, `context_skills`

**New surface:**

Standard profile (7 tools):
- `remember` - store observations (memory)
- `learn` - assert claims with evidence (knowledge)
- `believe` - declare commitments (wisdom)
- `recall` - unified search with `include_hypotheses` flag
- `trace` - provenance chain
- `link` - typed relationships
- `patterns` - workflow templates (renamed from context_skills)

Reasoning profile adds 5:
- `reason`, `reflect`, `hypothesize`, `revise`, `commit`

Internal only (SAGE/admin):
- `context_admin`, `context_accept_belief`, `context_reject_belief`

**Infrastructure:**
- YAML config at `src/context_service/config/mcp_tools.yaml`
- Profile-based registration via `register_profile_tools()`
- `MCP_TOOL_PROFILE` env var for profile selection
- Version bumped 1.3.0 -> 1.3.1
