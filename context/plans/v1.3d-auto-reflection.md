Implement automatic reflection triggers for confidence shifts, contradictions, and high-uncertainty queries.

## Existing Infrastructure

| Component | Location | Status |
|-----------|----------|--------|
| `create_auto_reflection` | `engine/auto_reflection.py` | Exists |
| `AutoReflectConfig` | `config/settings.py` | Exists (has `enabled`, `on_supersession`, `on_revision`) |
| MetaObservation model | `engine/models.py` | Exists |

## Scope

| Trigger | Condition | Action |
|---------|-----------|--------|
| Confidence shift | Belief confidence changes > threshold | Call `create_auto_reflection` |
| Contradiction | New claim contradicts existing fact | Flag for reflection, call `create_auto_reflection` |
| High uncertainty | Query returns low-confidence results | Annotate response with `reflection_suggested: true` |

## Design Decisions (locked)

| Question | Decision | Rationale |
|----------|----------|-----------|
| Sync vs async | Async (background) | Avoid write latency; use existing Dagster sensor pattern |
| Rate limiting | Max 10 reflections per silo per hour | Prevent reflection spam |
| Thresholds | `confidence_shift_threshold: 0.2`, `uncertainty_threshold: 0.4` | Tunable in config |
