## Decision 1: Where does transitivity inference run?

| Option | Pros | Cons |
|--------|------|------|
| **A. Query-time (virtual)** | Always fresh, no invalidation needed | Blows 250ms target on dense graphs, O(N^3) at depth 3 |
| **B. Dagster asset (batch)** | Predictable latency, can run after extraction | Stale until next run, needs invalidation on source change |
| **C. Custodian worker** | Integrated with existing validation flow | Adds load to custodian, may delay promotion |

**Recommendation:** B (Dagster asset) - safest for latency, runs after extraction asset, can tombstone stale edges.

## Decision 2: Storage model for inferred edges

| Option | Pros | Cons |
|--------|------|------|
| **A. Materialized** (`inferred=true` flag) | Fast queries, explicit in graph | Needs invalidation when source edges change |
| **B. Virtual** (computed at query time) | Always consistent | Expensive traversal, silo isolation tricky |

**Recommendation:** A (Materialized) with invalidation handled by the Dagster asset.

## Decision 3: Confidence decay formula

When A->B (conf=0.9) and B->C (conf=0.8), what is A->C confidence?

| Option | Formula | Result |
|--------|---------|--------|
| **A. Multiplicative** | 0.9 * 0.8 | 0.72 |
| **B. Minimum** | min(0.9, 0.8) | 0.80 |
| **C. Weighted average** | (0.9 + 0.8) / 2 | 0.85 |
| **D. Source-aware** | Multiplicative but dedupe same-source | varies |

**Recommendation:** A (Multiplicative) - standard for independent probabilities. Add source-deduplication later if needed.

## Decision 6: Extraction confidence vs edge confidence

Two different things:
- **Extraction confidence:** LLM's certainty it correctly identified a causal claim in text
- **Edge confidence:** Belief that the causal relationship is actually true

| Option | Model |
|--------|-------|
| **A. Single field** | Use extraction confidence directly as edge confidence |
| **B. Dual fields** | `extraction_confidence` and `consensus_confidence` separate |
| **C. Promotion flow** | Extraction confidence -> custodian validation -> consensus confidence |

**Recommendation:** C (Promotion flow) - aligns with existing Claim->Fact flow.
