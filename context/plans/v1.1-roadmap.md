## Dependency Graph

```
T3 (Belief Synthesis)
  └── Pattern Nodes (patterns emerge from belief clusters)
        └── T4 (Belief Revision) (revision triggered by pattern shifts)

T6 (Session Compaction)
  └── (standalone, no deps)

Causal Edges
  └── (standalone, extraction vocabulary addition)

Auto-Reflection
  └── T2 supersession (already wired)
  └── T4 revision (needs T4 first for revision triggers)
```

## Sequencing

**Phase 1 (v1.1-a):** T3 Belief Synthesis + T6 Session Compaction (parallel)
**Phase 2 (v1.1-b):** Pattern Nodes + Causal Edges (parallel)
**Phase 3 (v1.2):** T4 Belief Revision (depends on patterns)
**Phase 4 (v1.3):** Auto-Reflection

## Open Questions (cross-cutting)

1. **Belief identity:** How do we match an incoming fact cluster to an existing Belief for revision vs creating a new Belief?
2. **Session boundaries:** What defines "session end" for T6 compaction in MCP (stateless per-request)?
3. **Cross-silo patterns:** Can patterns span silos, or are they silo-scoped like everything else?
4. **Agent identity in reflection:** Are auto-generated reflections attributed to a system agent or the triggering agent?
