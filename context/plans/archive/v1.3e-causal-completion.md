## Implementation Notes

- `invalidate_derived_edges` lives in `engine/causal_invalidation.py` (not in the Dagster asset file) so it can be imported without triggering the Dagster runtime.
- `split_belief` and `SplitBeliefResult` added to `engine/revision.py`. Child beliefs are linked to the parent via `:REVISED_FROM` edges. Parent belief is NOT auto-staled — callers decide whether to supersede.
- Cypher queries `_FIND_DERIVED_EDGES` and `_TOMBSTONE_DERIVED_EDGE` use soft-delete (`invalidated = true`) rather than physical removal.
