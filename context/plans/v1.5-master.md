## Phase Details

### 5.0 Auth Fix

**Problem:** `getattr(auth, "agent_id", None)` always returns None.

**Scope:**
- Add `agent_id: str | None` to `AuthContext`
- Parse `X-Agent-Id`, `X-Parent-Agent-Id`, `X-Agent-Role` headers in MCP server
- Dev fallback: `settings.dev_agent_id` (new setting, defaults to `"dev-agent"`)
- Remove all `getattr(auth, "agent_id", None)` guards

**Location:** `auth/context.py`, `mcp/server.py`, `config/settings.py`

### 5a Agent Nodes + SPAWNED_BY

**Scope:**
- New `:Agent` node in graph (agent_id, role, lineage_root_id, silo_id, created_at)
- `SPAWNED_BY` edge type in primitives
- `upsert_agent()` in engine protocols
- Validate parent exists in same silo before writing edge
- Denormalize `lineage_root_id` for O(1) ancestry queries
- Wire into `context_reason`, `context_reflect`

**Constraints:**
- 3-hop max lineage depth
- All queries include silo boundary check

**Location:** `engine/queries.py`, `engine/protocols.py`, `engine/memgraph_store.py`, `primitives/schema/edges.py`

### 5b Reflection Filtering

**Scope:**
- Add `agent_id` filter parameter to `context_get_reflections`
- Add index on `MetaObservation(agent_id)`
- Update queries to support optional agent_id predicate

**Location:** `mcp/tools/context_reflect.py`, `db/indexes.py`, `db/queries.py`

### 5c Chain Continuity

**Scope:**
- Add optional `parent_chain_id` to `context_reason`
- Write `CONTINUES` edge when provided
- Traverse `CONTINUES` in `context_provenance`

**Open question:** CONTINUES (sequential handoff) vs DERIVED_FROM (evidence-based). May need both.

**Location:** `mcp/tools/context_reason.py`, `services/context.py`

### 5d Per-Silo Config

**Scope:**
- `SiloConfig` model with tunable parameters
- Store in graph as properties on `:Silo` node or separate `:SiloConfig` node
- Configurable: retention policies, heat decay rates, validator thresholds
- Defaults from global settings, overridable per silo

**Location:** `models/silo.py` (new), `services/silo.py`, `config/settings.py`

## Deferred to v1.6

- Multi-silo per org
- Cross-silo metacognition
- Agent quota/rate limiting per agent_id
- Agent management API (create, delete, list agents)

## Open Questions

1. **WorkOS agent_id source:** Custom attribute vs derived from user_id?
2. **Caller-supplied override:** Allow orchestrators to pass worker identity without separate token?
3. **Custodian vote counting:** Should custodian chains count toward R2 consensus?

## Success Criteria

- `agent_id` resolves to real value on every MCP call
- Agent nodes created on first write, SPAWNED_BY edges link parent/child
- Lineage queries bounded at 3 hops, use denormalized root for fast path
- Reflection filter works without breaking unfiltered queries
- Chain continuity enables cross-agent reasoning provenance
- Per-silo config overrides global defaults
- All phases pass `just check`
