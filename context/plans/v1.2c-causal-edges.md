**Status:** Blocked (design decisions required)
**Branch:** v1.2c-causal-edges
**Created:** 2026-05-03
**Priority:** P0 if partners want causal reasoning, P2 otherwise

Extract and reason over causal relationships in the knowledge graph.

## Scope

From v1.2-backlog.md:

1. **Extraction prompts** - Update to elicit CAUSES/CORROBORATES edges
2. **Transitivity inference** - A CAUSES B CAUSES C implies A CAUSES C
3. **Confidence aggregation** - Multi-source corroboration boosts confidence
4. **Negative causation** - PREVENTS edge type

## Dependencies

- `CITEEdgeType` in primitives already has CAUSES, CORROBORATES
- Need to check if PREVENTS exists or needs adding

## Design Questions

Before implementation:

1. **Where does inference run?**
   - Custodian worker (batch)?
   - Query-time (on-demand)?
   - Dagster asset?

2. **Transitivity depth limit?**
   - Unbounded could explode
   - Suggest max 3 hops

3. **Confidence decay on transitive edges?**
   - A->B (0.9) + B->C (0.8) = A->C (0.72)?
   - Or different formula?

4. **Storage model for inferred edges?**
   - Materialized with `inferred=true`?
   - Virtual (computed at query time)?
