## Phase Details

### 4.0 Quick Fixes

**Heat backwards compat:**
- Handle missing `event_type` field in heat asset (default to "read")
- Location: `src/context_service/pipelines/assets/heat.py`

**Silo API cleanup:**
- Remove `silo_create` MCP tool (misleading - implies multi-silo)
- Auto-create silo on first use in any context_* tool
- Update `silo_list` to return the single org silo
- Document that multi-silo is v1.5+
- Location: `src/context_service/mcp/tools/silo.py`, `src/context_service/services/silo.py`

### 4a Belief Merging

When Leiden clustering produces overlapping clusters, beliefs synthesized from each may conflict or duplicate. This phase handles merging.

**Scope:**
- Detect belief overlap (same subject, similar content across clusters)
- Merge strategy: union facts, reconcile confidence, preserve provenance
- Update `engine/synthesis.py` to handle merge cases
- Add `MERGED_FROM` edge type to track lineage

**Out of scope:**
- Cross-silo belief merging (v1.5)

### 4b Partial Revision + Cascade Flagging

When new evidence partially contradicts a belief, split rather than fully supersede.

**Partial revision:**
- Detect partial overlap (some claims still valid)
- Split belief into revised + retained portions
- Update confidence on retained portion

**Cascade flagging:**
- When Belief A references Belief B and B is revised, flag A for review
- Add `revision_cascade_pending` property
- Custodian sensor to process flagged beliefs

**Depends on:** 4a (belief merging semantics inform split logic)

### 4c Multi-Chain Sessions + Auto-Close

Build on v1.3c session compaction to support multiple reasoning chains per session.

**Multi-chain sessions:**
- Session can contain multiple reasoning chains
- `context_reason` accepts optional `session_id` to group chains
- Cross-chain `REFERENCES` edges within session

**Auto-close timeout:**
- Sessions auto-close after configurable timeout (default 30min)
- Dagster sensor checks for stale open sessions
- Triggers compaction on close

**Location:** `src/context_service/engine/compaction.py`, new `engine/sessions.py`

### 4d Hierarchical Meta-Memory

Meta-observations about other meta-observations (meta-meta).

**Scope:**
- `context_reflect` can target another MetaObservation node
- Query support for reflection depth
- Retention: meta-observations don't decay (per spec)

**Use case:** Agent notices a pattern in its own reflections ("I keep second-guessing decisions about X")

**Out of scope:**
- Cross-silo metacognition (v1.5)
- Agent identity inheritance (v1.5)
