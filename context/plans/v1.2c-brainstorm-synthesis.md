## Summary

Causal edges require 8 design decisions before implementation. The analysis reveals tight coupling between decisions and a clear implementation order. Key insight: query-time inference is incompatible with latency targets; materialized storage with Dagster batch inference is the only viable path.

## Final Recommendations

| # | Decision | Recommendation | Rationale |
|---|----------|----------------|-----------|
| 1 | Where inference runs | **Dagster asset** | Query-time blows 250ms target; fits existing pipeline |
| 2 | Storage model | **Materialized** (`inferred=true`) | Fast reads, rollback via bulk delete |
| 3 | Confidence formula | **Multiplicative + source-dedup** | Correct for independent chains; dedup prevents double-counting |
| 4 | Depth limit | **3 hops** (configurable) | Math: 0.8^2 = 0.64 clears min confidence floor |
| 5 | PREVENTS | **Add to primitives NOW** | Blocker; extraction will corrupt data without it |
| 6 | Confidence fields | **Dual fields** (`extraction_confidence`, `consensus_confidence`) | Matches Claim->Fact flow |
| 7 | Feature flags | **Three separate flags** (extraction, inference, query) | Surgical rollback per failure mode |
| 8 | Backfill | **New only + coverage_epoch** | Transparent partial coverage |

## Key Tradeoffs Accepted

1. **Staleness over latency** - Batch inference means inferred edges are stale until next asset run. Acceptable for probabilistic chains.

2. **Storage cost over consistency** - Materialized edges grow O(N^2) on dense silos but enable sub-250ms queries.

3. **Complexity now over debt later** - Three flags, dual confidence fields, source-dedup all add upfront complexity but prevent hard-to-diagnose production issues.

4. **Partial coverage over expensive backfill** - Existing silos get sparse causal graphs; transparency via epoch marking is cheaper than re-extraction.
