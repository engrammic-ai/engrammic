# v1.3b LLM-Detected Patterns

**Status:** Complete
**Parent:** [v1.3-master.md](./v1.3-master.md)
**Depends on:** [v1.3a-pattern-detection.md](./v1.3a-pattern-detection.md)
**Created:** 2026-05-03
**Est:** 1-2 days

Add LLM-based pattern detection for flexible, semantic patterns that rule-based detection misses.

## Scope

Piggybacks on v1.3a pattern infrastructure. Adds LLM classification step.

| Component | Description |
|-----------|-------------|
| Pattern prompt | Prompt template for pattern identification |
| Batch processor | Dagster asset to run LLM over candidate clusters |
| Confidence calibration | Map LLM confidence to pattern confidence |

## Design Decisions (locked)

| Question | Decision | Rationale |
|----------|----------|-----------|
| Which LLM? | Haiku | Cost-sensitive batch job; Sonnet overkill for classification |
| Batch size | 50 clusters per run | Balance throughput vs memory |
| Scheduling | Daily, after clustering asset | Needs fresh clusters |

## Tasks

1. Design pattern detection prompt (input: cluster facts, output: pattern type + confidence)
2. Implement `llm_pattern_detection` Dagster asset
3. Wire to pattern infrastructure from v1.3a (`engine/patterns.py`)
4. Add cost/latency guardrails (max tokens, timeout)
5. Gate behind `llm_pattern_enabled` flag
6. Tests

## Fail-Safe Behavior

| Failure Mode | Behavior |
|--------------|----------|
| LLM timeout | Skip batch, log warning, continue next cycle |
| LLM error rate > 10% | Circuit breaker trips, disable for 1 hour |
| Confidence < 0.3 | Discard pattern (hallucination filter) |
| Cost > daily budget | Pause until next day |
