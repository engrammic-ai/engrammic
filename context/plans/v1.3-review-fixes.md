## Phase 1: P0 Critical (Immediate)

### 1.1 Prompt Injection Fixes (AI-001/002/003) - S effort each

**Problem:** User content interpolated via `.format()` without sanitization enables prompt injection.

**Files:**
- `src/context_service/extraction/filter/llm_classifier.py:74-76`
- `src/context_service/extraction/service.py:91-93`
- `src/context_service/clustering/service.py:315-317`

**Fix:** Create a shared sanitization utility, wrap user content in XML data tags:

```python
# src/context_service/llm/sanitize.py (new)
def escape_for_prompt(text: str) -> str:
    """Escape braces and wrap in data tags to prevent injection."""
    escaped = text.replace("{", "{{").replace("}", "}}")
    return f"<data>{escaped}</data>"
```

Then update each location to use it before `.format()`.

### 1.2 Cross-Silo Data Leak (B-001) - S effort

**Problem:** `CREATE_MEMBER_OF` and `BATCH_CREATE_MEMBER_OF` match nodes without `silo_id` constraint.

**File:** `src/context_service/db/queries.py:109-114, 200-206`

**Fix:** Add `silo_id: $silo_id` to node MATCH clauses:
```cypher
-- Before
MATCH (n {id: $node_id})
-- After  
MATCH (n {id: $node_id, silo_id: $silo_id})
```

### 1.3 Qdrant/Memgraph Desync (E-001) - M effort

**Problem:** If Qdrant upsert fails after Memgraph write succeeds, node is graph-reachable but invisible to vector search.

**File:** `src/context_service/services/context.py:187-212`

**Fix:** Wrap Qdrant upsert in try/except. On failure:
1. Log error with node_id
2. Delete the Memgraph node (rollback)
3. Re-raise to signal failure to caller

### 1.4 DELETE_CLUSTERS Count Bug (B-002) - S effort

**File:** `src/context_service/db/queries.py:101-105`

**Fix:** Count before DETACH DELETE, return the count.
