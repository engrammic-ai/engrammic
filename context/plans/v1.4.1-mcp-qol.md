# v1.4.1 MCP Quality of Life

**Status:** Draft
**Created:** 2026-05-03
**Goal:** Fix broken auth fields, simplify tool surface, reduce agent friction

## Problem Statement

Current MCP tools have unnecessary friction:
- `silo_id` required on every call but derivable from auth
- `agent_id` always None (AuthContext field missing)
- `session_id` always None (same issue)
- 15 tools with overlapping functionality
- Agents must call `silo_list` before any write

|-------|-------|-----|--------|

## Phase Details

### 4.1.0 Fix AuthContext

**Problem:** `getattr(auth, "agent_id", None)` always returns None.

**Changes:**
- Add `agent_id: str | None = None` to `AuthContext`
- Add `session_id: str | None = None` to `AuthContext`
- Populate from headers (`X-Agent-Id`) or derive from `user_id`
- Default `agent_id` to `f"user:{auth.user_id}"` when not provided
- Default `session_id` from WorkOS token hash

**Files:** `auth/context.py`, `mcp/server.py`

### 4.1.1 Make silo_id Optional

**Current:** Every tool requires `silo_id`, then validates it matches `derive_silo_id(org_id)`.

**Change:** Make `silo_id` optional, default to derived value.

```python
# Before
async def context_remember(silo_id: str, content: str, ...) -> dict:

# After  
async def context_remember(content: str, ..., silo_id: str | None = None) -> dict:
    resolved_silo = silo_id or str(derive_silo_id(auth.org_id))
```

**Files:** All 15 tools in `mcp/tools/`

### 4.1.2 Consolidate Read Tools

Merge redundant tools into enhanced core tools:

| Drop | Merge into | How |
|------|------------|-----|
| `context_belief_history` | `context_history` | Add `include_confidence_trend` param |
| `context_get_reflections` | `context_get` | Add `include_reflections` param |
| `context_provenance` | `context_graph` | Add `mode="provenance"` or preset edge types |

**Result:** 15 - 3 = 12 tools

### 4.1.3 Consolidate Write Tools (New Surface)

Add new simplified tools alongside existing:

```python
context_store(
    content: str,
    layer: Literal["memory", "knowledge", "wisdom", "intelligence", "meta"],
    evidence: list[str] | None = None,  # required for knowledge
    about: list[str] | None = None,     # required for wisdom/meta
    steps: list[dict] | None = None,    # required for intelligence
    tags: list[str] | None = None,
    session: str | None = None,
)

context_recall(
    query: str,
    node_ids: list[str] | None = None,  # if present, fetch by ID
    depth: int = 0,                      # 0 = flat, 1-3 = graph
    layers: list[str] | None = None,
    top_k: int = 10,
    as_of: str | None = None,
)

context_link(
    from_id: str,
    to_id: str,
    relationship: str,
)

context_admin(
    action: Literal["silo_list", "close_session", "provenance", "history"],
    ref: str | None = None,
    name: str | None = None,
)
```

**Result:** 4 new simplified tools

### 4.1.4 Remove Old Tools

- Remove old tools once new surface is validated
- Update any internal tests that reference old tool names
- No deprecation period - clean cut

- `session_id` stable within WorkOS session
- Agent can write without calling `silo_list` first
